# Technical Assessment

You have been provided with a basic test suite to test [The NearForm Login Page](https://nearform.github.io/tester-assessment/).

## Expectations

As a remote company, we place high value on clear communication. Please explain your thought process and the reasoning behind your actions. Aim to write maintainable and clean tests that you can be proud of.

## Business Requirements

The following are the business requirements for testing the login page:

- The email address must follow the format `<name>@<domain>.com`, `.co.uk`, or `.it`.
- The password should be at least 8 characters long and contain a combination of upper and lower case letters, numbers, and at least one special character.
- If the password or email is in an incorrect format, a validation message should be displayed.
- Users should be able to validate their credentials by either using the SUBMIT button or pressing ENTER.
- Upon entering a valid username and password, users should be directed to a screen displaying a welcome message.

## Notes

Since this is solely a frontend assessment, there are no correct details stored in a database. Therefore, the system will accept any valid credentials that meet the specified business requirements.
